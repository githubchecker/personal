## 🗺️ Stage 0 — Concept Map

How does a file of **random numbers** (0.0.1) become a smart model? This lesson is the **training
loop** — the process that sets the "knobs." It deepens 0.0.1 (parameters), 0.0.4 (lifecycle), and
0.0.7 (the transformer). You don't implement it, but the vocabulary (*loss, gradient descent,
backprop, epoch, learning rate*) shows up in interviews and when you fine-tune later (Phase 4).

## 🔑 New Terms (plain English)

- **Forward pass** — running an input through the model to get a guess.
- **Loss** — a single number for *how wrong* the guess was (lower = better). The common one is
  **cross-entropy**.
- **Backpropagation** — working backwards to find *which way* to nudge each parameter to reduce loss.
- **Gradient** — the "direction + size" of that nudge for each parameter.
- **Optimizer** — the algorithm that applies the nudges (commonly **Adam**).
- **Learning rate** — how big each nudge is.
- **Epoch / step** — one full pass over the data / one batch update.

## 🎈 Stage 1 — The Simple Idea (analogy: archery practice)

Learning is **archery practice**. You shoot an arrow (the model **guesses** the next word), see how
far you missed the bullseye (**loss**), work out which muscles to adjust (**backpropagation** finds
the **gradient**), make a **small** correction (**optimizer** × **learning rate**), and shoot again
— millions of times until you reliably hit the target.

**The "Aha!":** training is just *guess → measure the miss → nudge every knob a tiny bit to miss
less → repeat*, billions of times.

## ⚙️ Stage 2 — How It Actually Works (the loop)

1. **Setup.** Take a batch of text; hide the next word. Input = the text so far, target = the real
   next word.
2. **Forward pass.** The (currently random) model produces **logits** — a score for every word
   (see 0.0.5). Early on, the right word scores poorly.
3. **Loss (the grade).** **Cross-entropy** checks the probability given to the *correct* word. High
   probability → low loss; low probability → high loss.
4. **Backpropagation (the blame game).** Work backwards from the loss to compute the **gradient** for
   every parameter: "to lower the loss, nudge this one up a little, that one down a lot."
5. **Optimizer (the update).** Adam applies the nudges:
   `new = old − (learning_rate × gradient)`. The model gets *very slightly* better.
6. **Repeat.** A **step** = one batch; an **epoch** = one full pass over the data. Do this trillions
   of times; loss falls from "gibberish" to "fluent."

**Learning rate matters:** too high → parameters swing wildly and loss **explodes** (often `NaN`);
too low → training crawls.

## 🚀 Stage 3 — In Practice / Why It Matters (awareness)

You won't write this loop — **pre-training** does it on the whole internet for millions of dollars,
and you start from the finished model. But the words appear in **fine-tuning** (Phase 4 LoRA adjusts
a *tiny* slice of parameters with exactly this loop) and in interviews. Knowing it also explains
*why* fine-tuning needs example data and can "forget" old skills if over-done.

## 🧠 Common Misconceptions

- **"Training and inference are the same."** → No — at inference the loss, backprop, and optimizer
  are **gone**; only the frozen forward pass runs (see 0.0.1 / Model Usage).
- **"More epochs are always better."** → No — too many and the model **overfits** (memorises instead
  of generalising).
- **"As an AI engineer I'll train base models."** → No — you orchestrate frozen models; at most you
  run a small fine-tune (Phase 4).

## 📌 Quick Reference (key facts)

- loop: **forward → loss → backprop → optimizer → repeat.**
- update rule: **new = old − (learning_rate × gradient).**
- **cross-entropy** = the usual loss; **Adam** = the usual optimizer.
- **epoch** = one pass over data; **step** = one batch.
- learning rate too high → loss explodes (`NaN`); too low → crawls.

> 🎯 **Interview angle:** "What does `loss.backward()` do?" → it runs **backpropagation** to compute
> the **gradients** (the nudge direction/size) for every parameter; the optimizer then applies them.

## 🛑 STOP — Self-Check

If the **learning rate** is set far too high, what happens to the loss during training, and why?

<details><summary>Answer</summary>

The **loss explodes** (gets worse, often becoming `NaN`). A huge learning rate makes each update
overshoot the target — `new = old − (huge × gradient)` swings parameters wildly past the good
values, so instead of settling toward low loss, the model bounces around and destabilises. Training
needs **small** steps to converge.
</details>
