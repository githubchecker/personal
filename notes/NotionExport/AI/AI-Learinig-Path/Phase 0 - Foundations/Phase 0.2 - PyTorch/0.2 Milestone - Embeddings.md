## 📄 `similarity.py` (fully commented)

```python
from sentence_transformers import SentenceTransformer, util

# 1. Load a small, fast embedding model (downloads once, then cached).
model = SentenceTransformer("BAAI/bge-small-en-v1.5")

# 2. Some sentences: two about dogs (should be similar), one unrelated.
sentences = [
    "A dog runs across the park",     # 0
    "A puppy plays outside on grass",  # 1  (close in meaning to 0)
    "The stock market fell today",     # 2  (unrelated)
]

# 3. Encode -> a tensor of shape (3, 384): 3 vectors, each 384 numbers.
#    convert_to_tensor=True keeps results as PyTorch tensors for the math below.
embeddings = model.encode(sentences, convert_to_tensor=True)
print("shape:", embeddings.shape)   # torch.Size([3, 384])

# 4. COSINE SIMILARITY measures the angle between two vectors:
#    1.0 = identical meaning, ~0 = unrelated, -1 = opposite.
#    util.cos_sim returns a 3x3 grid comparing every pair.
scores = util.cos_sim(embeddings, embeddings)

print(f"dog vs puppy   : {scores[0][1]:.3f}")   # high  (similar)
print(f"dog vs market  : {scores[0][2]:.3f}")   # low   (different)
```

## ▶️ Run it

```powershell
python similarity.py
```

Expect the **dog↔puppy** score to be clearly higher than **dog↔market** — meaning, captured as
numbers, then compared with geometry.

## 🧠 Why cosine, not plain distance?
Cosine similarity compares **direction** (the "topic" of the vector), ignoring length. Two
sentences about dogs point the same way regardless of how long they are, so cosine is the standard
choice for semantic search.
