## 🗺️ Stage 0 — Concept Map

PyTorch is the library that does the heavy number-crunching behind AI models. Its single most
important object is the **tensor**. This is the first PyTorch lesson because *everything* — an
embedding, a model's weights, an image — is stored as a tensor. It unlocks the rest of Module 0.2
(Hugging Face, practical PyTorch) and explains why embeddings (0.3.2) are "just lists of numbers."

## 🎈 Stage 1 — The Simple Idea (analogy: a smart spreadsheet of numbers)

A **tensor** is a container for numbers arranged in a grid — like a spreadsheet, but it can have
any number of dimensions:
- 0 dimensions = a single number (a *scalar*), e.g. `7`
- 1 dimension = a list (a *vector*), e.g. `[1, 2, 3]`
- 2 dimensions = a table (a *matrix*), e.g. rows and columns
- 3+ dimensions = stacks of tables (e.g. a batch of images)

**The "Aha!":** a tensor is just a multi-dimensional array of numbers — *plus* the ability to run
on a GPU and do fast math. AI is mostly multiplying big tensors together very quickly.

## ⚙️ Stage 2 — How It Actually Works

Install first: `pip install torch`.

```python
import torch   # the PyTorch library

# 1. Make tensors from Python lists.
scalar = torch.tensor(7)             # 0-D
vector = torch.tensor([1, 2, 3])     # 1-D, shape (3,)
matrix = torch.tensor([[1, 2],
                       [3, 4]])      # 2-D, shape (2, 2)

# 2. SHAPE = the size along each dimension. The first thing you check about any tensor.
print(vector.shape)   # torch.Size([3])
print(matrix.shape)   # torch.Size([2, 2])

# 3. DTYPE = the kind of number stored (e.g. float32, int64). Controls precision & memory.
print(matrix.dtype)   # torch.int64

# 4. MATH is element-wise by default; matrix multiply uses @ (or torch.matmul).
print(vector + 10)        # tensor([11, 12, 13])
print(matrix @ matrix)    # matrix multiplication

# 5. DEVICE = where the tensor lives: "cpu" or "cuda" (NVIDIA GPU). Move with .to(...).
print(vector.device)      # cpu
# vector_gpu = vector.to("cuda")   # only if a GPU is available
```

Three properties define every tensor:
- **shape** — its dimensions, e.g. `(2, 2)`.
- **dtype** — the number type, e.g. `float32` (4 bytes) vs `float16` (2 bytes).
- **device** — `cpu` or `cuda` (GPU). GPU math is dramatically faster for large tensors.

## 🚀 Stage 3 — In Practice / Why It Matters

When you embed a sentence in Phase 2, the result is a 1-D tensor like `shape (1536,)` — 1536
floats capturing meaning. A model's "knowledge" (its parameters from Phase 0.0) is millions of
these floats in big tensors. Knowing shape, dtype, and device lets you read model cards and debug
the #1 PyTorch error class: shape mismatches.

**Most common beginner mistake:** multiplying tensors whose shapes don't line up (e.g. a `(2,3)`
times a `(2,3)` with `@`). PyTorch raises a shape error. The fix: print `.shape` before any matrix
op and confirm the inner dimensions match. The reason: matrix multiply requires the columns of
the first to equal the rows of the second.

## 🛑 STOP — Self-Check

What are the **shape** and **dtype** of `torch.tensor([[1.0, 2.0, 3.0]])`, and why is the shape
`(1, 3)` rather than `(3,)`?

<details><summary>Answer</summary>

Shape **`(1, 3)`**, dtype **`float32`** (the `.0` decimals make them floats). The extra outer
brackets `[[ ]]` make it a 2-D tensor with **1 row and 3 columns**, not a flat 1-D vector of 3.
The number of bracket levels = the number of dimensions.
</details>
